## Scope

Adds the deferred "when" dimension to survey targeting (v2 shipped who=audience +
where=placement in migration 025), plus the post-purchase render surface.

## 1. Date-window targeting

- Migration (next free number at build time — 027 is customer-support billing, so likely
  028): `surveys` gains `starts_at timestamptz null`, `ends_at timestamptz null`.
  Null = no bound (current behaviour).
- Eligibility: `pickSurveyFor` (src/lib/survey.ts) filters `starts_at <= now() < ends_at`
  server-side. Add to the PURE `pickEligibleSurvey` as a `now` param so it stays
  unit-testable. RLS "surveys: public select active" can stay as-is (window enforced in
  app code) or be tightened to the window in the same migration — decide at build time;
  app-side is enough since out-of-window surveys simply aren't offered.
- Admin UI: survey create/edit modal gains optional "Show from / until" datetime-local
  fields; survey row shows a "scheduled" / "ended" chip when outside the window.

## 2. After-N-days targeting

- Same migration: `surveys.min_account_age_days int null` (null = immediate).
  Interpretation: only show to users whose profiles.created_at is at least N days old —
  Danny's "show this survey to people who've been around a while" case.
- Eligibility: needs profile.created_at — add to AudienceSignals (getAudienceSignals
  already queries with the service client; one more select).
- Admin UI: optional number field "Only show to accounts older than N days".

## 3. Post-purchase placement surface (blocked on payments)

- The `post_purchase` placement enum + admin option already exist (migration 025);
  nothing renders it. When the Stripe checkout success page/flow is built, call
  `pickSurveyFor(service, rls, userId, 'post_purchase')` on the post-checkout
  confirmation surface and render `<SurveyCard/>` — identical wiring to the account page
  (see src/app/app/account/page.tsx).
- The `first_purchase` audience also starts matching real users at that point
  (purchases.status='complete' count == 1) — no code change needed.
